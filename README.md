## Features / Routes

- POST : `/greet/:person`
  - Parameters
    - must include a person as a route parameter.
    - must include a text property on the request body.
  - Response
    - status 200, and string body if parameters look good.
      - body: `TEXT PERSON`
    - status 400, if no text is present.
